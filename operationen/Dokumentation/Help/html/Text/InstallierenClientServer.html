Installieren für mehrere Benutzer an einer zentralen Stelle

Verzeichnisse als vertrauenswürdig einstellen
Richten Sie das Programmverzeichnis und das Verzeichnis für die gemeinsamen Daten ein wie unter Verzeichnisse als vertrauenswürdig einrichten beschrieben.

Wählen Sie diese Installationsart, wenn

  • Sie das Programm mit mehreren Personen benutzen, aber auf dieselben Daten zugreifen möchten
  • Sie das Programm nur an einer Stelle installieren möchten und dieses eine Programm von verschiedenen PCs aus starten wollen

Voraussetzungen für die Installation

  • Sie müssen Administrator sein und Lese- und Schreibrechte in dem Verzeichnis haben, in dem das Programm installiert ist.
  • Sie müssen Administrator sein und Lese- und Schreibrechte in dem Verzeichnis haben, in dem die gemeinsamen Daten liegen.

Voraussetzungen zum Starten des Programmes

  • Sie müssen Leserechte in dem Verzeichnis haben, in dem das Programm installiert ist.
  • Sie müssen Lese- und Schreibrechte in dem Verzeichnis für die gemeinsamen Daten haben.

Hinweis

  • Sobald eine neue Version installiert ist, steht diese allen zur Verfügung.
  • Achten Sie bei der Serverinstallation darauf, dass Sie die UNC-Pfade so eingeben, wie Sie vom Client aus später angegeben werden. Siehe hierzu auch Verzeichnisse als vertrauenswürdig einrichten.
  • Anstelle eines UNC-Pfades können Sie auch einen Laufwerksbuchstaben verwenden, z.B. 'L:\Logbuch' anstelle von '\\cmaurer\Logbuch'.

Installation

  1. Führen Sie die Installationsart '3a. Mehrere Benutzer, eine zentrale Installation ... Programm' aus und installieren Sie das Programm einmal auf einem Server, wobei Sie die UNC-Pfade so angeben, wie sie von den Clients aus verwendet werden. Hierbei wird die Datei addTrust.bat erstellt, die bei einer Client-Installation ('3b. Mehrere Benutzer ... Verknüpfung') ausgeführt wird.
  2. Führen Sie auf jedem Client die Installationsart '3b. Mehrere Benutzer, eine zentrale Installation ... Verknüpfung' aus.

Beispiel für eine Installation

Einmal führen Sie folgendes aus, um das Programm an einer zentralen Stelle zu installieren:

  1. Starten Sie die Installation

    Das Programm soll in \\cmaurer\Logbuch installiert werden. Die gemeinsamen Daten sollen in \\cmaurer\LogbuchDaten installiert werden.
    Hierbei ist cmaurer der Rechnername und Logbuch und LogbuchDaten sind freigegebene Verzeichnisse (shared folders). Auch wenn Sie direkt auf dem Rechner cmaurer arbeiten, geben Sie NICHT einen lokalen Pfad wie 'c:\Programme\OP-LOG' an, sondern den UNC-Pfad \\cmaurer\Logbuch, der von allen Clients aus gültig ist.

    Angenommen, \\cmaurer\Logbuch steht allen Benutzern zur Verfügung. Sie installieren das Programm in \\cmaurer\Logbuch und die gemeinsamen Daten in \\cmaurer\LogbuchDaten

  2. Wählen Sie '3a. Mehrere Benutzer, eine zentrale Installation ... Programm' aus

  3. Geben Sie als Programmverzeichnis \\cmaurer\Logbuch ein.
  4. Geben Sie als Datenbankverzeichnis \\cmaurer\LogbuchDaten ein.

  5. Schließen Sie die Installation ab.

  6. Hierdurch wird die Datei \\cmaurer\Logbuch\addTrust.bat angelegt mit folgendem Inhalt:

    c:
    cd %windir%\Microsoft.NET\Framework\v2.0.50727
    caspol.exe -q -machine -addgroup "All_Code" -url "\\cmaurer\Logbuch\*" FullTrust
    caspol.exe -q -machine -addgroup "All_Code" -url "\\cmaurer\LogbuchDaten\*" FullTrust

    7. Stellen Sie sicher, dass nach der Installation in dieser Datei die UNCPFade so stehen wie im Beispiel abgegeben. Wenn Sie als nächstes auf dem Client bei der Installation eine Verknüpfung anlegen lassen wie im folgenden beschrieben, wird addTrust.bat ausgeführt und muss die richtigen UNC-Pfade enthalten.

Auf jedem PC, auf dem das Programm gestartet werden soll, tun Sie folgendes:

  1. Wählen Sie '3b. Mehrere Benutzer, eine zentrale Installation ... Verknüpfung' aus

  2. Geben Sie als Programmverzeichnis \\cmaurer\Logbuch ein.
  3. Geben Sie als Datenbankverzeichnis \\cmaurer\LogbuchDaten ein.

  4. Schließen Sie die Installation ab.

Wenn Sie das Programm auf dem Client nicht starten können, weil die Datei addTrust.bat nicht die richtigen Pfade enthält, tun Sie folgendes auf dem Client:

  1. Legen eine Verknüpfung an, die auf \\cmaurer\Logbuch\operationen.exe verweist:


     
  2. Führen Sie folgendes Skript aus, wodurch die Verzeichnisse \\cmaurer\Logbuch und \\cmaurer\LogbuchDaten dem .NET Framework als vertrauenswürdig mitgeteilt werden.

    Dieser Schritt ist notwendig, da Sie das Programm sonst nicht starten können.
    c:
    cd %windir%\Microsoft.NET\Framework\v2.0.50727
    caspol.exe -q -machine -addgroup "All_Code" -url "\\cmaurer\Logbuch\*" FullTrust
    caspol.exe -q -machine -addgroup "All_Code" -url "\\cmaurer\LogbuchDaten\*" FullTrust

    Passen Sie das Skript an, wenn Sie andere Verzeichnisse als \\cmaurer\Logbuch und \\cmaurer\LogbuchDaten verwenden.
  3. Starten Sie das Programm nun durch Doppelklicken auf die Verknüpfung.
