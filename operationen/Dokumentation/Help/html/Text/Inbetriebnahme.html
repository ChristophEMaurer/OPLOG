Inbetriebnahme

Programminstallation

Laden Sie das Programm herunter wie hier angegeben und installieren Sie das Programm
  • Für einen einzelnen Benutzer
  • Für mehrere Benutzer auf mehreren PCs
  • Für mehrere Benutzer an einer zentralen Stelle (Serverinstallation)

Seriennummern / Lizenzdatei

Geben Sie Seriennummern ein oder installieren Sie eine Lizenzdatei.

Benutzer anlegen

Legen Sie hier alle benötigen Benutzer an. In diesem Fenster legen Sie auch fest, ob ein Benutzer Weiterbilder ist. Ein Weiterbilder hat Zugriff auf alle Daten der Benutzer, die ihm zugeordnet sind. Lesen Sie hierzu auch Rollen und Rechte.

Abteilungen, Weiterbilder, Rollen und Rechte

Die Rechte sind vom Programm vorgegeben, diese können Sie nicht ändern.

Legen Sie die Rollen fest. Ordnen Sie den Rollen die entsprechenden Rechte zu. Ordnen Sie die Benutzer den Rollen zu. Ordnen Sie die Benutzer den Weiterbildern zu.

Legen Sie die Abteilungen an und ordnen Sie die Benutzer den Abteilungen zu.

Facharztgebiete, Weiterbildungsrichtlinien und Zuordnungen von Weiterbildungsrichtlinien zu OPSKodes

Legen Sie die Facharztgebiete (wie Allgemeinchirurgie, Herzchirurgie), an.

Geben Sie die Weiterbildungsrichtlinien für ein Facharztgebiet manuell ein oder importieren Sie diese. Eine Weiterbildungsrichtlinie für das Facharztgebiet Basischirurgie ist z.B. "50 Ultraschalluntersuchungen bei chirurgischen Erkrankungen und Verletzungen".

Damit Operationen automatisch einer Weiterbildungsrichtlinie zugeordnet werden können, muss festgelegt werden, welcher OPSKode zu welcher Weiterbildungsrichtlinie gehört. Geben Sie die Zuordnungen von OPSKodes zu Weiterbildungsrichtlinien manuell ein oder importieren Sie diese.

Datensicherung einrichten

Es ist wichtig, regelmäßig eine Datensicherung durchzuführen. Ob eine Datensicherung korrekt durchgeführt wurde, weiß man erst, wenn man die Daten aus einer Sicherung erfolgreich wiederhergestellt hat. Sichern Sie Ihre Daten!

Im laufenden Betrieb

Operationen importieren

Sie können Operationen manuell eingeben oder importieren. Kontrollieren Sie die Daten im Fenster "Auswertungen > Liste aller durchgeführten Prozeduren".

Jedes Jahr gibt es einen neuen OPS-Katalog. Dieser wird vom DIMDI herausgegeben. Im Jahr 2010 war dieser unter http://www.dimdi.de > Webshop zu erhalten:

(1 x OPS Version 2010, Systematisches Verzeichnis, EDV-Fassung als ASCII-Datei, Artikel-Nr. KD10014, Format: ASCII-Datei, 20,00 EUR).

Die OPSKodes aus dem OPSKatalog können Sie ansehen oder ändern. Der Zweck des OPSKataloges ist, dass man in anderen Fenster einen OPSKode auswählen kann - wobei man OPSKode und OPSText sieht - und diesen nicht manuell eingeben muss.

Wenn Sie einen neuen OPSKatalog importieren wollen, tun Sie folgendes:

  1. Löschen Sie den aktuellen OPSKatalog, indem Sie unter "Offizielle Dokumente > OPS-Katalog" die Schaltfläche "Gesamten OPS-Katalog löschen" klicken.
  2. Importieren Sie den neuen OPS-Katalog, indem Sie "Datenimport > OPS-Katalog" anklicken und die entsprechende DIMDI-Datei auswählen.

Wenn Sie einen DIMDI OPS-Katalog importieren, ohne den vorhandenen zu löschen, ist das kompliziert. Manchmal ändert sich nur die Rechtschreibung - z.B. ss statt ß aufgrund der neuen Deutschen Rechtschreibung. Sie müssen für jeden Eintrag entscheiden, ob der vorhandene beibehalten werden soll, ob ein vorhandener OPSText durch einen veränderten ersetzt werden soll, und welche neuen Einträge hinzugefügt werden sollen.

Tipp: Wenn Sie beim Datenimport einschalten, dass nicht vorhandene Operationen automatisch eingefügt werden, so werden diese in den OPS-Katalog eingefügt, falls Sie dort fehlen. Damit kommen genau die OPS-Kodes im OPS-Katalog vor, die bei Ihnen durchgeführt werden.

Neue Programmversion einspielen

Unter "Extras > Auf Update aus dem Internet überprüfen" können Sie eine neue Programmversion einspielen, sobald diese vorhanden ist.

ACHTUNG: Wenn eine neue Version eingespielt wird, und die die Datenbankstruktur verändert, so müssen alle Benutzer, die auf dieselbe Datenbank zugreifen, ihr Programm ebenfalls aktualisieren.

ACHTUNG: Bei einer neuen Programmversion können neue Rechte eingeführt werden. Diese müssen Sie unter "Verwaltung > Rollen und Rechte" den entsprechenden Rollen zuordnen, ansonsten bleibt eine neuer Menüpunkt für alle Benutzer ausgegraut.

Nicht Zugeordnete Operationen kontrollieren

Kontrollieren Sie, welche Operationen nicht durch Zuordnungen von OPSKodes zu Weiterbildungsrichtlinien erfasst werden. Dieses sehen Sie im Fenster Auswertungen > Weiterbildungsstand > Einzelübersicht

History-Tabelle kontrollieren

Alle Aktionen werden festgehalten, indem Sie in eine Änderungs-Historie eingetragen werden. Diese Tabelle wächst stetig an. Wenn Sie zuviele Einträge enthält, können Sie einige hieraus löschen. Die Anzahl der Einträge sehen Sie unter "Verwaltung > System > Übersicht".

Rechte kontrollieren

Kontrollieren Sie ab und zu die Rechte der Benutzer. Wenn man nicht als Benutzer einer Gruppe selber angemeldet ist, ist es schwierig zu wissen, welche Rechte der Benutzer wirklich hat. Eine Übersicht hierzu sehen Sie unter "Verwaltung > Rollen und Rechte > Übersicht".
